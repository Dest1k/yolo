# Запуск YOLO Detector на Raspberry Pi 5 — полная инструкция с нуля

Инструкция написана максимально подробно, по шагам. Если что-то непонятно —
делай ровно как написано, по порядку, не пропуская шаги.

---

## 0. Что у тебя есть и что это вообще такое

Твой одноплатник — **Raspberry Pi 5** (мини-компьютер размером с банковскую
карту). На нём нет ни операционной системы, ни диска — всё это нужно записать
самому на карту памяти. По фото видно, что слот для карты пустой, значит система
ещё нигде не стоит.

Наш проект (YOLO Detector) — это приложение, которое распознаёт объекты на видео
в реальном времени и рисует вокруг них рамки. На Pi мы запускаем **десктопную**
версию (оконное приложение).

---

## 1. Что нужно докупить (одной платы и type-C кабеля НЕ хватит)

> ⚠️ Важно: на Raspberry Pi 5 порт USB-C — **только для питания**. Подключиться
> к компьютеру «по этому кабелю напрямую» и зайти в плату нельзя. Связь будет по
> Wi-Fi или сетевому кабелю.

| Что купить / найти | Зачем | Подсказка |
|---|---|---|
| **microSD карта, 16 ГБ и больше** | На неё ставится операционная система. Без неё плата не включится во что-либо полезное. | Класс 10 / маркировка A1 или A2. 32 ГБ — комфортно. |
| **Блок питания USB-C на 5V / 5A (27 Вт)** | Кабель сам по себе — не питание. Нужен зарядник. | Лучше всего официальный Raspberry Pi 27W. Подойдёт хороший зарядник 5V/3A, но возможны предупреждения о нехватке тока. |
| **Второй компьютер + картридер для microSD** | Чтобы записать систему на карту. | Любой Windows / Mac / Linux. Картридер часто встроен в ноутбук или есть USB-переходник. |
| **Wi-Fi или сетевой кабель (Ethernet)** | Чтобы зайти на плату и скачать проект. | Wi-Fi встроен в Pi 5 — настроим при записи карты. |
| **Монитор с кабелем micro-HDMI + USB-клавиатура/мышь** ИЛИ **программа VNC** | Наше приложение — это ОКНО. В голой текстовой консоли оно не запустится: ему нужен «рабочий стол». | У Pi 5 разъёмы **micro-HDMI** (маленькие), нужен соответствующий кабель/переходник. Альтернатива — удалённый рабочий стол по VNC (см. шаг 9). |
| **Источник видео** | Чтобы было что распознавать. | USB-вебкамера (самый простой вариант) ИЛИ MJPEG-поток (например, твой телефон с нашим Android-приложением). |

## 2. Записываем операционную систему на карту памяти

Делается на **втором компьютере**, не на Pi.

1. Открой сайт **raspberrypi.com/software** и скачай программу **Raspberry Pi
   Imager**. Установи её.
2. Вставь microSD-карту в картридер второго компьютера.
3. Запусти Raspberry Pi Imager. В нём три кнопки:
   - **Choose Device** (Выбрать устройство) → выбери **Raspberry Pi 5**.
   - **Choose OS** (Выбрать систему) → выбери **Raspberry Pi OS (64-bit)**.
     ВАЖНО: именно 64-bit.
   - **Choose Storage** (Выбрать носитель) → выбери свою карту памяти.
     ⚠️ Перепроверь, что выбрал именно карту, а не диск компьютера — всё на ней
     будет стёрто.
4. Нажми **Next** (Далее). Программа спросит: «Применить настройки?» — нажми
   **Edit Settings** (Редактировать настройки) и заполни:
   - **Hostname** (имя устройства): например `yolopi`
   - **Username и Password**: придумай логин и пароль. **ЗАПИШИ их** — они
     понадобятся, чтобы зайти на плату.
   - **Configure wireless LAN** (настроить Wi-Fi): впиши имя своей Wi-Fi сети
     (SSID), пароль от неё и страну (например `RU`).
   - На вкладке **Services**: поставь галочку **Enable SSH** → вариант **Use
     password authentication**.
5. Сохрани настройки и нажми **Yes / Write**. Дождись окончания записи (несколько
   минут) и сообщения, что карту можно извлечь. Вынь карту.

---

## 3. Первый запуск Pi

1. Вставь записанную microSD-карту в Pi (слот снизу платы, подписан SD CARD).
2. Если используешь монитор — подключи его кабелем micro-HDMI к разъёму **рядом с
   USB-C** (это HDMI0), а также подключи USB-клавиатуру и мышь.
3. Подключи питание в порт **USB-C**.
4. Первая загрузка длится примерно 1–2 минуты. Если есть монитор — дождись
   появления рабочего стола.

---

## 4. Подключаемся к плате (если работаешь без монитора — по сети)

Если у тебя есть монитор/клавиатура — открой на плате приложение **Terminal**
(чёрный значок с `>_`) и переходи к шагу 5.

Если работаешь удалённо (без монитора), на **втором компьютере** открой терминал
и введи (подставь свой логин и имя из шага 2):

```bash
ssh пользователь@yolopi.local
```

Например: `ssh pi@yolopi.local`. При первом подключении согласись (`yes`) и введи
пароль.

> Если `yolopi.local` не находится — зайди в свой Wi-Fi роутер, найди там IP-адрес
> платы (что-то вроде `192.168.0.42`) и подключайся по нему:
> `ssh пользователь@192.168.0.42`

---

## 5. Обновляем систему

В терминале платы по очереди:

```bash
sudo apt update
sudo apt full-upgrade -y
```

(Пароль вводится «вслепую» — символы не отображаются, это нормально.)
Может занять несколько минут. По завершении на всякий случай перезагрузись:

```bash
sudo reboot
```

После перезагрузки снова подключись по SSH (шаг 4).

## 6. Ставим Java и Git

Нашему проекту нужны Java 17 и Git:

```bash
sudo apt install -y openjdk-17-jdk git
```

Проверь, что Java встала:

```bash
java -version
```

Должна показаться строка с `17`.

## 7. Скачиваем проект

```bash
git clone https://github.com/dest1k/yolo.git
cd yolo
```

Теперь ты находишься в папке проекта.

> Поддержка ARM-плат (Raspberry Pi и аналогов) уже встроена в проект: сборка сама
> подтягивает правильные библиотеки под процессор Pi. Ничего вручную править не
> нужно.

---

## 8. Включаем «рабочий стол» для удалённого запуска (если без монитора)

Наше приложение — оконное, ему нужен графический рабочий стол. Если у тебя
подключён монитор — этот шаг пропусти, переходи к шагу 10.

Если работаешь по SSH без монитора — включи VNC (удалённый рабочий стол):

```bash
sudo raspi-config
```

В меню стрелками: **Interface Options** → **VNC** → **Yes** (включить). Затем
**Finish**. Перезагрузись, если попросит.

Дальше на **втором компьютере** установи **RealVNC Viewer** (скачивается с
realvnc.com) и подключись к адресу `yolopi.local` (или IP платы) — увидишь
рабочий стол Pi. Дальнейшие команды (шаг 10) запускай в терминале уже **внутри
этого рабочего стола**, а не в SSH-сессии.

---

## 9. Подключаем камеру

Самый простой вариант — воткнуть **USB-вебкамеру** в любой USB-порт платы. В
приложении она будет под индексом `0`.

Альтернатива без вебкамеры — использовать **MJPEG-поток** (например, телефон с
нашим Android-приложением, у которого включён MJPEG-сервер). Тогда вместо индекса
камеры в приложении укажешь адрес вида `http://АЙПИ-ТЕЛЕФОНА:8080`.

---

## 10. Собираем и запускаем приложение

В терминале (на рабочем столе Pi или по VNC), находясь в папке `yolo`:

```bash
./gradlew :desktop:run
```

Первый запуск долгий — Gradle скачивает зависимости (нужен интернет) и собирает
проект. Это может занять 10–20 минут на Pi. Наберись терпения. В следующий раз
будет намного быстрее.

Когда соберётся — откроется окно приложения.

---

## 11. Настройки в приложении

1. **Тип модели (Model type):** выбери **ONNX** — так проще всего на Pi.
2. **Browse:** укажи файл модели `.onnx`. Рекомендую самую лёгкую — **yolov10n**
   или **yolov8n**. Где взять — см. ниже «Где взять модель».
3. **Источник видео (Video source):**
   - `0` — если подключена USB-вебкамера;
   - либо `http://...` — адрес MJPEG-потока (например, с телефона).
4. **GPU mode:** выбери **CPU**. У Raspberry Pi нет CUDA/DirectML, ускорения тут
   нет — работаем на процессоре.
5. **Input size (размер входа):** поставь поменьше — **320**. Чем меньше, тем
   быстрее (но чуть менее точно).
6. Нажми **Start**.

---

## 12. Где взять модель .onnx

Самый простой путь — экспортировать готовую модель на **втором компьютере**, где
есть Python:

```bash
pip install ultralytics
python -c "from ultralytics import YOLO; YOLO('yolov10n.pt').export(format='onnx', imgsz=320, simplify=True)"
```

Получится файл `yolov10n.onnx`. Перекинь его на Pi (через флешку, или командой
`scp yolov10n.onnx пользователь@yolopi.local:~/`), и укажи его в шаге 11.

---

## 13. Чего ожидать по скорости

Raspberry Pi 5 — слабый по меркам нейросетей. Инференс идёт на процессоре (без
видеоускорения), поэтому реалистично — **несколько кадров в секунду** на лёгкой
модели (nano) с маленьким входом (320). Для экспериментов, наблюдения и трансляции
по сети — нормально. Плавных 30 кадров/с ждать не стоит.

Что помогает ускориться:
- модель **nano** (yolov10n / yolov8n), не больше;
- **input size 320** (или даже 256);
- закрыть лишние программы на плате.

---

## 14. Если что-то пошло не так

- **`./gradlew: Permission denied`** → выполни `chmod +x gradlew` и повтори.
- **Сборка падает на скачивании зависимостей** → проверь интернет на плате:
  `ping -c3 github.com`. Нужен доступ в сеть.
- **Окно приложения не появляется, ошибка про дисплей/`HeadlessException`** →
  значит нет рабочего стола. Подключи монитор или включи VNC (шаг 8) и запускай
  внутри рабочего стола.
- **Камера не открывается (индекс 0)** → проверь, что вебкамера видна системе:
  `ls /dev/video*` (должно что-то показать). Попробуй индекс `1`.
- **`yolopi.local` не находится по SSH** → используй IP-адрес платы из роутера.
- **Предупреждение о низком напряжении / молнии на экране** → слабый блок питания,
  возьми 5V/5A.
