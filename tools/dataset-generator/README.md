# Генератор синтетического датасета (FLUX + LLM + авторазметка)

Собирает детекторный датасет с нуля под **любой объект** (не только дроны): картинки рисует
**FLUX.1-schnell**, промпты к ним пишет локальная **LLM** (LM Studio или любой OpenAI-совместимый
эндпоинт), а боксы расставляет на выбор **YOLO-World / Grounding DINO / OWLv2** (zero-shot). На
выходе — готовый YOLO-датасет (`images/`, `labels/`, `dataset.yaml`), который сразу скармливается
обучалкам из соседних сайдкаров.

```
tools/dataset-generator/
  generate_dataset.py       движок (config-driven, без интерактива) — можно гонять headless
  generate_dataset_gui.py   окно: все параметры + редактор словаря промптов + живой лог + прогресс
```

Всё **само ставится и качается**: недостающие пакеты (`pip`) и модели (FLUX, веса разметчиков с
HuggingFace) подтягиваются при первом запуске, если включена галка «Сам ставить…» (по умолчанию).

## Три фазы (включаются по отдельности)

1. **Промпты** — LLM пишет большой набор уникальных сцен, сбалансированных по масштабу объекта
   **дефицит-планировщиком** (точные пропорции при любом размере датасета). Пишутся в
   `prompts.jsonl` построчно, прогон возобновляется.
2. **Картинки** — FLUX.1-schnell рендерит их с квантизацией трансформера (`torchao` fp8 / `nf4`
   4-bit / `layerwise`), суперчанками; результат стримится на диск. Либо режим **«свои картинки»**
   (`generation.backend = own`) — генерация пропускается, берётся готовая папка.
3. **Разметка** — выбранный open-vocab детектор даёт боксы по списку синонимов класса → YOLO `.txt`
   + `dataset.yaml`.

## Графический запуск (рекомендуется)

```bash
python tools/dataset-generator/generate_dataset_gui.py
```
Вкладки и живой лог с полосой прогресса снизу:

| Вкладка | Что задаёт |
|---|---|
| **Запуск** | пути (с «Обзор»), число картинок, движок генерации (`flux`/`own`), какие фазы, авто-установка, Старт/Стоп, «Сохранить конфиг…» |
| **LLM** | base URL, API-ключ, модель, температура, max tokens, тумблер LM Studio (`lms load/unload`) |
| **Картинки (FLUX)** | quant mode, батчи, шаги, guidance, размер, JPEG, префикс файлов, repo FLUX для авто-скачивания, суффикс, TF32 |
| **Промпты** | **универсальный редактор словаря**: объект + любые свои категории (`## Имя` и варианты ниже); **микс масштабов** (`вес \| фраза`); шаблон system-prompt |
| **Разметка** | **выбор движка** (YOLO-World / Grounding DINO / OWLv2) с пояснениями, веса/модели, синонимы, conf/iou, id и имена классов |

Окно только пишет JSON-конфиг и запускает движок — конфиг, сохранённый из окна, точно так же
гоняется headless на GPU-машине. Значения запоминаются (`~/.dataset_generator_gui.json`).

> Окну нужен только Python с **tkinter** (Windows — из коробки; Linux — `sudo apt install
> python3-tk`). Тяжёлые зависимости импортирует и ставит движок при запуске.

## Запуск из консоли

```bash
python generate_dataset.py                 # встроенные дефолты
python generate_dataset.py my_config.json  # свой конфиг
GEN_CONFIG=my_config.json python generate_dataset.py
```

## Универсальный словарь промптов

Вся «соль» — на вкладке **Промпты** (или в секции `prompts` конфига):

- `object_noun` — что генерируем (любой объект);
- `categories` — **произвольные** категории вариативности: имя → список вариантов. В окне
  редактируется блоками: строка `## Имя категории`, ниже её варианты по одному в строке. LLM в
  каждом батче фокусируется на случайном варианте из каждой категории (через `{config_block}`);
- `objects` — список объектов сцены для генерации (по одному в строке). Если их **2+**, часть
  картинок будет содержать НЕСКОЛЬКО разных объектов сразу: доля таких сцен = `multi_object_prob`
  (0..1), максимум объектов в сцене = `multi_object_max`. Пусто → используется один `object_noun`;
- `object_scales` — строки `вес | фраза`; веса это пропорции, планировщик держит их точно;
- `system_template` — шаблон запроса к LLM с плейсхолдерами `{batch_size}`, `{object_noun}`,
  `{config_block}`.

Связка с разметкой: имена объектов сцены обычно совпадают с именами классов на вкладке
«Разметка». Тогда сцены с 2+ объектами автоматически размечаются мультиклассово — каждый объект в
свой класс.

Под свой объект просто перепиши `object_noun`, категории и `labeling.classes`/`class_names` —
пайплайн от предметной области не зависит.

## Производительность и полнота утилизации (RTX 5090 / 128 ГБ / Ultra 9 285K)

Дефолты подобраны, чтобы машина грузилась полностью:

- **FLUX**: `quant_mode=torchao` (нативный fp8 на Blackwell), `micro_batch=16`, `encode_batch=128`,
  `super_chunk=2000`. Сохранение JPEG вынесено в пул потоков (`generation.save_workers=12`), чтобы
  GPU не ждал диск (заодно грузит ядра CPU). При нехватке VRAM — уменьшить `micro_batch`.
- **Разметка** идёт **батчами** (`labeling.batch=16`) на GPU, а не по одной картинке — это
  главный фактор полной загрузки GPU в фазе 3.
- **Авто-сплит train/val**: `val_split` (доля в val, по умолчанию 0.15) — встроен в генератор.
  Создаёт `train.txt`/`val.txt` со списками картинок (без перемещения файлов) и прописывает их в
  `dataset.yaml`. `0` — без сплита.
- **Негативы (пустые сцены)**: `prompts.empty_scene_prob` — доля картинок чистого фона без
  объектов (пустые метки), снижает ложные срабатывания. По умолчанию 0 (выкл).

## Выбор движка разметки

`labeling.backend`:

- **`yoloworld`** — быстрый, лёгкий (ultralytics, веса `yolov8x-worldv2.pt`). Дефолт, хорош для
  большинства классов.
- **`groundingdino`** — точнее на редких/необычных классах и сложных описаниях, но тяжелее и
  медленнее (transformers, модель `IDEA-Research/grounding-dino-base`).
- **`owlv2`** — ещё один open-vocab (Google, `google/owlv2-base-patch16-ensemble`), иногда лучше
  ловит мелкие объекты.

### Несколько классов (мультикласс)

Классы задаются **группами**: каждый класс — своя группа синонимов, id = порядок (0,1,2…). Разные
классы НЕ смешиваются в одну кучу. В окне (вкладка «Разметка») это блоки:

```
## drone
drone
quadcopter
uav
## bird
bird
flying bird
```

В конфиге это `labeling.classes` = `[{"name":"drone","synonyms":[…]}, {"name":"bird","synonyms":[…]}]`.
Все синонимы всех классов идут в словарь open-vocab модели, а каждая найденная рамка получает id
своего класса. `dataset.yaml` тоже пишется со всеми именами. Веса/модели любого движка скачиваются
автоматически на первом запуске.
